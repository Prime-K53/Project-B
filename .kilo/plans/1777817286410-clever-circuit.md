# Plan: Convert Prime ERP to Offline Electron Desktop App

## Project Overview
Convert the current web-based Prime ERP (Vite + Express + SQLite) into a 100% offline desktop application using Electron.

## Current State
- **Frontend**: Vite React app (port 5173)
- **Backend**: Express + SQLite (port 3000)
- **No Electron**: Currently runs as web app requiring separate processes

## Goal
A single .exe file that bundles the frontend, backend, and SQLite database, running completely offline.

---

## Implementation Plan

### Phase 1: Project Preparation

#### 1.1 Update Frontend API Configuration
- Modify `frontend/config/api.js` to detect Electron environment
- Fallback to `http://localhost:3000/api` in dev, use IPC in production
- Add electron detection: check for `window.electronAPI`

#### 1.2 Configure Vite for Production Build
- Update `frontend/vite.config.ts`:
  - Change default port from 3005 to 5173
  - Ensure build outputs to `dist/` folder
  - Remove dev-only proxy settings for production build

#### 1.3 Test Frontend Production Build
- Run `npm run build` in frontend directory
- Verify `frontend/dist/` contains index.html and assets

---

### Phase 2: Electron Setup

#### 2.1 Add Electron Dependencies
Add to root `package.json`:
```json
{
  "main": "electron/main.js",
  "scripts": {
    "electron": "electron .",
    "build:electron": "electron-builder --win --x64"
  },
  "build": {
    "appId": "com.primeerp.workspace",
    "productName": "Prime ERP",
    "directories": { "output": "release" },
    "files": ["electron/**/*", "backend/**/*", "frontend/dist/**/*"],
    "extraResources": [
      { "from": "backend", "to": "backend" },
      { "from": "frontend/dist", "to": "frontend/dist" }
    ],
    "win": { "target": [{ "target": "nsis", "arch": ["x64"] }], "icon": "electron/icon.ico" },
    "nsis": { "oneClick": false, "allowToChangeInstallationDirectory": true }
  },
  "devDependencies": {
    "electron": "^33.3.1",
    "electron-builder": "^25.1.8"
  }
}
```

#### 2.2 Create Electron Main Process (`electron/main.js`)
- Import: `electron`, `path`, `child_process`, `fs`
- Functions:
  - `startBackend()`: Spawn Express server as child process
  - `createWindow()`: Create BrowserWindow with proper config
  - Menu: File, Edit, View, Window with standard actions
- IPC handlers:
  - `get-backend-url`: Return localhost:3000
  - `get-app-version`: Return app version

#### 2.3 Create Preload Script (`electron/preload.js`)
- Expose `electronAPI` to renderer via contextBridge
- Methods: `getBackendUrl()`, `getAppVersion()`, `getUserDataPath()`

---

### Phase 3: Production Build

#### 3.1 Backend Considerations
- SQLite database location: Use `app.getPath('userData')` for data storage
- Ensure database file persists between sessions
- Backend runs on port 3000 (hardcoded in main.js)

#### 3.2 Build Commands
```bash
# Install dependencies
npm install electron electron-builder --save-dev

# Build frontend
npm run build --prefix frontend

# Package Electron app
npm run build:electron
```

#### 3.3 Output
- `.exe` installer in `release/` folder
- Can also generate portable `.exe`

---

### Phase 4: Testing & Verification

#### 4.1 Development Testing
- Run `npm run electron` in root
- Verify backend starts with app
- Verify frontend loads in Electron window
- Test core functionality: login, navigation, data operations

#### 4.2 Production Testing
- Run the built .exe
- Verify offline functionality
- Test database persistence
- Test PDF generation works

## Risks & Considerations

1. **SQLite path**: Ensure database writes to user data directory, not app directory (which is read-only in production)
2. **Port conflicts**: If port 3000 is busy, backend will fail - need error handling
3. **PDF generation**: Ensure puppeteer/pdf libraries work in Electron context
4. **Large bundle size**: Electron adds ~150MB; acceptable for desktop app
